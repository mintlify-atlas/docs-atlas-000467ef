---
title: "fields.markdoc()"
description: "Stores rich text content as a Markdoc (.mdoc) file with a WYSIWYG editor."
---

The `markdoc` field is a rich text editor that reads and writes content in [Markdoc](https://markdoc.dev/) format. It stores content in a separate `.mdoc` file (or inline when using `fields.markdoc.inline()`). It is the recommended field for rich text in new projects.

## Import

```typescript
import { fields } from '@keystatic/core';
```

## Usage

<CodeGroup>

```typescript basic
richText: fields.markdoc({
  label: 'Rich text',
})
```

```typescript with options
content: fields.markdoc({
  label: 'Content',
  description: 'The main body of the post.',
  options: {
    bold: true,
    italic: true,
    heading: [1, 2, 3],
    image: {
      directory: 'public/images/posts',
      publicPath: '/images/posts/',
    },
  },
})
```

```typescript with content components
content: fields.markdoc({
  label: 'Content',
  components: {
    // See the Content Components docs
  },
})
```

</CodeGroup>

## Options

<ParamField path="label" type="string" required>
  The label shown above the field in the Admin UI.
</ParamField>

<ParamField path="description" type="string">
  Helper text shown below the label in the Admin UI.
</ParamField>

<ParamField path="extension" type="'mdoc' | 'md'">
  The file extension used for the content file. Defaults to `'mdoc'`. Set to `'md'` to use plain Markdown file extensions.
</ParamField>

<ParamField path="components" type="Record<string, ContentComponent>">
  Custom content components that can be inserted into the editor. See the [Content Components](/concepts/content-components) docs for details.
</ParamField>

<ParamField path="options" type="MarkdocEditorOptions">
  Controls which formatting toolbar items are shown in the editor. All options default to `true` unless specified.

  <ParamField path="options.bold" type="boolean">
    Enable bold text. Defaults to `true`.
  </ParamField>

  <ParamField path="options.italic" type="boolean">
    Enable italic text. Defaults to `true`.
  </ParamField>

  <ParamField path="options.strikethrough" type="boolean">
    Enable strikethrough text. Defaults to `true`.
  </ParamField>

  <ParamField path="options.code" type="boolean">
    Enable inline code. Defaults to `true`.
  </ParamField>

  <ParamField path="options.heading" type="boolean | number[] | { levels: boolean | number[]; schema: Record<string, ComponentSchema> }">
    Enable headings. Pass `true` to allow all levels (1–6), or an array such as `[1, 2, 3]` to restrict to specific levels. Pass an object with `levels` and `schema` to attach additional fields to heading nodes. Defaults to `true`.
  </ParamField>

  <ParamField path="options.blockquote" type="boolean">
    Enable blockquotes. Defaults to `true`.
  </ParamField>

  <ParamField path="options.orderedList" type="boolean">
    Enable ordered (numbered) lists. Defaults to `true`.
  </ParamField>

  <ParamField path="options.unorderedList" type="boolean">
    Enable unordered (bullet) lists. Defaults to `true`.
  </ParamField>

  <ParamField path="options.table" type="boolean">
    Enable tables. Defaults to `true`.
  </ParamField>

  <ParamField path="options.link" type="boolean">
    Enable hyperlinks. Defaults to `true`.
  </ParamField>

  <ParamField path="options.divider" type="boolean">
    Enable horizontal dividers. Defaults to `true`.
  </ParamField>

  <ParamField path="options.codeBlock" type="boolean | { schema: Record<string, ComponentSchema> }">
    Enable fenced code blocks. Pass `true` or an object with `schema` to attach extra fields (e.g., language, filename) to code block nodes. Defaults to `true`.
  </ParamField>

  <ParamField path="options.image" type="boolean | object">
    Controls inline image support inside the editor. Defaults to `true` (images enabled with default settings).

    <ParamField path="options.image.directory" type="string">
      The directory where images uploaded via the editor are stored. Defaults to the same directory as the entry.
    </ParamField>

    <ParamField path="options.image.publicPath" type="string">
      The URL prefix prepended to image filenames when the content is read. For example, `'/images/posts/'`.
    </ParamField>

    <ParamField path="options.image.transformFilename" type="(originalFilename: string) => string">
      A function to transform the filename of uploaded images.
    </ParamField>

    <ParamField path="options.image.schema" type="object">
      Extra fields attached to each image node. By default includes `alt` (text) and `title` (hidden text).
    </ParamField>
  </ParamField>
</ParamField>

## Reader API

When read via the Reader API, `fields.markdoc()` returns `{ node: MarkdocNode }` — the parsed Markdoc AST root node. You are responsible for rendering the AST.

```typescript
import { createReader } from '@keystatic/core/reader';
import keystaticConfig from './keystatic.config';

const reader = createReader(process.cwd(), keystaticConfig);
const post = await reader.collections.posts.read('my-post');
// post.content.node is a Markdoc RootNode
```

### Rendering Markdoc content

Use the `@markdoc/markdoc` package to render the AST to HTML or React:

```typescript
import Markdoc from '@markdoc/markdoc';

const post = await reader.collections.posts.read('my-post');
const html = Markdoc.renderers.html(
  Markdoc.transform(post.content.node)
);
```

### Using `format.contentField`

When a `markdoc` field is the `format.contentField` of a collection, Keystatic writes the Markdoc content as the body of the entry file instead of a separate file:

```typescript keystatic.config.ts
posts: collection({
  label: 'Posts',
  slugField: 'title',
  format: { contentField: 'content' },
  schema: {
    title: fields.slug({ name: { label: 'Title' } }),
    content: fields.markdoc({ label: 'Content' }),
  },
})
```

## Inline variant

`fields.markdoc.inline()` writes the Markdoc content inline with other YAML fields in the same file, rather than in a separate `.mdoc` file:

```typescript
someContent: fields.markdoc.inline({
  label: 'Some content',
})
```

The entry file will include the Markdoc text as a YAML block scalar:

```yaml
someContent: |
  # Title

  Some content
```

## In your config

```typescript keystatic.config.ts
import { config, collection, fields } from '@keystatic/core';

export default config({
  storage: { kind: 'local' },
  collections: {
    posts: collection({
      label: 'Posts',
      slugField: 'title',
      format: { contentField: 'content' },
      schema: {
        title: fields.slug({ name: { label: 'Title' } }),
        content: fields.markdoc({
          label: 'Content',
          options: {
            heading: [1, 2, 3],
            image: {
              directory: 'public/images/posts',
              publicPath: '/images/posts/',
            },
          },
        }),
      },
    }),
  },
});
```
