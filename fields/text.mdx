---
title: "fields.text()"
description: "Stores a text string value in your content schema."
---

The `text` field stores a plain text string. By default it renders a single-line `<input type="text">` in the Admin UI. Set `multiline: true` to render a `<textarea>` instead.

## Import

```typescript
import { fields } from '@keystatic/core';
```

## Usage

<CodeGroup>

```typescript basic
quote: fields.text({
  label: 'Quote',
})
```

```typescript multiline with validation
bio: fields.text({
  label: 'Bio',
  description: 'A short author biography.',
  multiline: true,
  validation: {
    isRequired: true,
    length: { min: 10, max: 500 },
  },
})
```

</CodeGroup>

## Options

<ParamField path="label" type="string" required>
  The label shown above the field in the Admin UI.
</ParamField>

<ParamField path="description" type="string">
  Helper text shown below the label in the Admin UI.
</ParamField>

<ParamField path="defaultValue" type="string | (() => string)">
  The initial value for the field when creating a new entry. Accepts a static string or a function that returns a string.

  Defaults to `''`.
</ParamField>

<ParamField path="multiline" type="boolean">
  When `true`, the Admin UI renders a `<textarea>` instead of a single-line text input. Useful for longer content such as descriptions or summaries.

  Defaults to `false`.
</ParamField>

<ParamField path="validation" type="object">
  Validation rules applied when saving an entry.

  <ParamField path="validation.isRequired" type="boolean">
    When `true`, the field must not be empty. Setting this to `true` also enforces a minimum length of `1`.
  </ParamField>

  <ParamField path="validation.length.min" type="number">
    The minimum number of characters the value must contain.
  </ParamField>

  <ParamField path="validation.length.max" type="number">
    The maximum number of characters the value may contain.
  </ParamField>

  <ParamField path="validation.pattern" type="{ regex: RegExp; message?: string }">
    A regular expression the value must match. Optionally provide a `message` to display when validation fails.
  </ParamField>
</ParamField>

## Reader API

The `text` field returns `string` in the Reader API.

```typescript
const post = await reader.collections.posts.read('my-post');
const title: string = post.title;
```

<Note>
  When `validation.isRequired` is `true`, the TypeScript type remains `string`. The field defaults to `''` and is never `null`.
</Note>

## In your config

```typescript keystatic.config.ts
import { config, collection, fields } from '@keystatic/core';

export default config({
  storage: { kind: 'local' },
  collections: {
    posts: collection({
      label: 'Posts',
      slugField: 'title',
      schema: {
        title: fields.slug({ name: { label: 'Title' } }),
        summary: fields.text({
          label: 'Summary',
          description: 'A brief description shown in post listings.',
          multiline: true,
          validation: {
            isRequired: true,
            length: { max: 160 },
          },
        }),
      },
    }),
  },
});
```
