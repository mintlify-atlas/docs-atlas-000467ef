---
title: "fields.mdx()"
description: "Stores rich text content as an MDX file with a WYSIWYG editor."
---

The `mdx` field is a rich text editor that reads and writes content in [MDX](https://mdxjs.com/) format. It stores content in a separate `.mdx` file (or inline when using `fields.mdx.inline()`). Choose `fields.mdx()` when your rendering pipeline is MDX-based and you need to use React components inside content.

## Import

```typescript
import { fields } from '@keystatic/core';
```

## Usage

<CodeGroup>

```typescript basic
richText: fields.mdx({
  label: 'Rich text',
})
```

```typescript with options
content: fields.mdx({
  label: 'Content',
  description: 'The main body of the post.',
  options: {
    bold: true,
    italic: true,
    heading: [1, 2, 3],
    image: {
      directory: 'public/images/posts',
      publicPath: '/images/posts/',
    },
  },
})
```

```typescript with content components
content: fields.mdx({
  label: 'Content',
  components: {
    // See the Content Components docs
  },
})
```

</CodeGroup>

## MDX limitations

### No import statements

Keystatic statically analyses MDX content, so `import` statements inside the MDX file are not supported.

```mdx
# This will NOT work
import { Card } from '../components/Card'

<Card title="Example" />
```

Instead, pass components to whatever renderer you use:

```tsx
import { Card } from '../components/Card';

<MdxRenderer components={{ Card }} />
```

### No raw HTML tags

Raw HTML tags inside MDX are not supported. Use Markdown equivalents:

```mdx
# Use this
[Keystatic website](https://keystatic.com)

# Not this
<a href="https://keystatic.com">Keystatic website</a>
```

## Reader API

When read via the Reader API, `fields.mdx()` returns the raw MDX content as a `string`. You are responsible for rendering it.

```typescript
import { createReader } from '@keystatic/core/reader';
import keystaticConfig from './keystatic.config';

const reader = createReader(process.cwd(), keystaticConfig);
const post = await reader.collections.posts.read('my-post');
// post.content is a string of MDX source
```

### Rendering MDX content

Use a library such as [next-mdx-remote](https://github.com/hashicorp/next-mdx-remote) or [mdx-bundler](https://github.com/kentcdodds/mdx-bundler) to compile and render the MDX string:

```tsx
import { MDXRemote } from 'next-mdx-remote/rsc';

export default async function PostPage({ params }) {
  const post = await reader.collections.posts.read(params.slug);
  return <MDXRemote source={post.content} />;
}
```

### Using `format.contentField`

When a `mdx` field is the `format.contentField` of a collection, Keystatic writes the MDX content as the body of the entry file instead of a separate file:

```typescript keystatic.config.ts
posts: collection({
  label: 'Posts',
  slugField: 'title',
  format: { contentField: 'content' },
  schema: {
    title: fields.slug({ name: { label: 'Title' } }),
    content: fields.mdx({ label: 'Content' }),
  },
})
```

## Inline variant

`fields.mdx.inline()` writes the MDX content inline with other YAML fields in the same file, rather than in a separate `.mdx` file:

```typescript
someContent: fields.mdx.inline({
  label: 'Some content',
})
```

The entry file will include the MDX text as a YAML block scalar:

```yaml
someContent: |
  # Title

  Some content
```

## In your config

```typescript keystatic.config.ts
import { config, collection, fields } from '@keystatic/core';

export default config({
  storage: { kind: 'local' },
  collections: {
    posts: collection({
      label: 'Posts',
      slugField: 'title',
      format: { contentField: 'content' },
      schema: {
        title: fields.slug({ name: { label: 'Title' } }),
        content: fields.mdx({
          label: 'Content',
          options: {
            heading: [1, 2, 3],
            image: {
              directory: 'public/images/posts',
              publicPath: '/images/posts/',
            },
          },
        }),
      },
    }),
  },
});
```
