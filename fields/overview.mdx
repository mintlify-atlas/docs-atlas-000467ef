---
title: "Fields API overview"
description: "Fields define the shape of your content. Use them inside the schema of collections and singletons."
---

Fields define what data each collection entry or singleton stores. You compose them inside the `schema` option of a `collection()` or `singleton()` call.

## Import

```typescript
import { fields } from '@keystatic/core';
```

All field helpers are exported from the `fields` object. There is no need to import individual field functions.

## Usage

```typescript keystatic.config.ts
import { config, collection, fields } from '@keystatic/core';

export default config({
  storage: { kind: 'local' },
  collections: {
    posts: collection({
      label: 'Posts',
      slugField: 'title',
      schema: {
        title: fields.slug({ name: { label: 'Title' } }),
        publishedOn: fields.date({ label: 'Published on' }),
        summary: fields.text({ label: 'Summary', multiline: true }),
        featured: fields.checkbox({ label: 'Featured' }),
      },
    }),
  },
});
```

## Reader API types

Each field produces a typed value when read with the Reader API. The table below summarises the return types for the fields covered on this tab.

| Field | Reader type |
|---|---|
| `text` | `string` |
| `integer` | `number \| null` |
| `number` | `number \| null` |
| `url` | `string \| null` |
| `slug` | `string` (the slug portion only) |
| `date` | `string \| null` (`YYYY-MM-DD`) |
| `datetime` | `string \| null` (`YYYY-MM-DDTHH:MM`) |

<Note>
  Fields marked with `validation.isRequired: true` narrow their Reader API type to exclude `null`.
</Note>
