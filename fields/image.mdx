---
title: "fields.image()"
description: "Stores an image file in your repository with configurable directory and public path."
---

The `image` field stores an image in your local file system or Git repository. In the Admin UI it renders an image picker. Images are committed directly to your repository alongside your content.

For cloud-hosted images managed outside your repository, see [`fields.cloudImage()`](/fields/cloud-image).

## Import

```typescript
import { fields } from '@keystatic/core';
```

## Usage

<CodeGroup>

```typescript basic
avatar: fields.image({
  label: 'Avatar',
})
```

```typescript with directory and publicPath
avatar: fields.image({
  label: 'Avatar',
  description: 'Profile picture for this author.',
  directory: 'public/images/avatars',
  publicPath: '/images/avatars/',
  validation: {
    isRequired: true,
  },
})
```

</CodeGroup>

## Image storage

By default, the `image` field creates a directory named after the entry slug and stores a file named after the field key inside it.

For example, given this config:

```typescript
authors: collection({
  schema: {
    avatar: fields.image({ label: 'Avatar' }),
  },
})
```

Creating a `john-doe` author entry and uploading a JPEG produces:

```
authors/
  john-doe/
    avatar.jpg
  john-doe.yaml
```

The value stored in `john-doe.yaml` is:

```yaml
avatar: avatar.jpg
```

### Configuring `directory`

Set `directory` to store images in a specific location, such as a `public` folder for static asset serving:

```typescript
avatar: fields.image({
  label: 'Avatar',
  directory: 'public/images/avatars',
})
```

With this config the `john-doe` JPEG is now stored at:

```
public/
  images/
    avatars/
      john-doe/
        avatar.jpg
authors/
  john-doe.yaml
```

The value in `john-doe.yaml` remains `avatar: avatar.jpg`. Set `publicPath` to control how this value is constructed.

### Configuring `publicPath`

`publicPath` sets the URL prefix that is prepended to the filename when writing the value to the entry data file. The entry slug and field name are composed with this prefix:

```typescript
avatar: fields.image({
  label: 'Avatar',
  directory: 'public/images/avatars',
  publicPath: '/images/avatars/',
})
```

With this config the `john-doe.yaml` value becomes:

```yaml
avatar: /images/avatars/john-doe/avatar.jpg
```

This is the value returned by the Reader API, so you can use it directly in an `<img src>` attribute.

## Options

<ParamField path="label" type="string" required>
  The label shown above the field in the Admin UI.
</ParamField>

<ParamField path="description" type="string">
  Helper text shown below the label in the Admin UI.
</ParamField>

<ParamField path="directory" type="string">
  The directory (relative to your project root) where the image file is stored. Defaults to the same directory as the entry.
</ParamField>

<ParamField path="publicPath" type="string">
  The URL prefix prepended to the image path when writing the value to the entry data file. Set this to the path from which your framework serves static assets (e.g., `'/images/avatars/'`). When omitted, only the bare filename is stored.
</ParamField>

<ParamField path="validation" type="object">
  Validation rules applied when saving an entry.

  <ParamField path="validation.isRequired" type="boolean">
    When `true`, saving without an image is not allowed.
  </ParamField>
</ParamField>

## Reader API

The `image` field returns `string | null` from the Reader API. The string is the full public path (prefix + filename), ready to use in an `<img src>` attribute.

```typescript
const author = await reader.collections.authors.read('john-doe');
// author.avatar is '/images/avatars/john-doe/avatar.jpg' or null
if (author.avatar) {
  return <img src={author.avatar} alt="Author avatar" />;
}
```

<Note>
  When `validation.isRequired` is `true`, the TypeScript type is `string` (never `null`). When it is `false` or omitted, the type is `string | null`.
</Note>

## In your config

```typescript keystatic.config.ts
import { config, collection, fields } from '@keystatic/core';

export default config({
  storage: { kind: 'local' },
  collections: {
    authors: collection({
      label: 'Authors',
      slugField: 'name',
      schema: {
        name: fields.slug({ name: { label: 'Name' } }),
        avatar: fields.image({
          label: 'Avatar',
          description: 'Profile picture shown on the author page.',
          directory: 'public/images/avatars',
          publicPath: '/images/avatars/',
          validation: { isRequired: true },
        }),
      },
    }),
  },
});
```
