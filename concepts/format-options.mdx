---
title: "Format options"
description: "Configure whether Keystatic stores content as YAML, JSON, or a single file with frontmatter."
---

Keystatic can store data in multiple formats: YAML, JSON, Markdoc (`.mdoc`), and MDX (`.mdx`). The `format` option on a collection or singleton controls which format is used.

By default, entries are stored as a YAML file (`index.yaml`). If the schema includes a `document`, `markdoc`, or `mdx` field, that field's content is written to a separate `.mdoc` or `.mdx` file alongside the data file.

---

## Default behavior (YAML + separate content file)

With this collection:

```typescript
blog: collection({
  label: 'Blog',
  slugField: 'title',
  schema: {
    title: fields.slug({ name: { label: 'Title' } }),
    publishedDate: fields.date({ label: 'Published date' }),
    body: fields.markdoc({ label: 'Body' }),
  },
})
```

Creating an entry with slug `what-a-day` produces:

```
blog/what-a-day/
├── index.yaml
└── body.mdoc
```

The `index.yaml` file:

```yaml
title: What a day
publishedDate: 2023-07-27
```

The `body.mdoc` file:

```markdown
What a **beautiful** day!

## Let's go to the beach

I say we pack our swimmers and towels and head to the beach.

Who's with me?
```

---

## JSON data (`format.data: 'json'`)

Use `format.data` to switch the data file from YAML to JSON:

```typescript
blog: collection({
  label: 'Blog',
  slugField: 'title',
  format: { data: 'json' },
  schema: { /* ... */ },
})
```

The generated files:

```
blog/what-a-day/
├── index.json
└── body.mdoc
```

The `index.json` file:

```json
{
  "title": "What a day",
  "publishedDate": "2023-07-27"
}
```

---

## Single-file output (`format.contentField`)

The `format.contentField` option lets you combine the data fields and the rich-text body into a single file. The metadata is placed in YAML frontmatter at the top, and the body content follows.

Set `contentField` to the key of a `markdoc`, `mdx`, or `document` field in your schema:

```typescript
blog: collection({
  label: 'Blog',
  slugField: 'title',
  format: { contentField: 'body' },
  schema: {
    title: fields.slug({ name: { label: 'Title' } }),
    publishedDate: fields.date({ label: 'Published date' }),
    body: fields.markdoc({ label: 'Body' }),
  },
})
```

Instead of two separate files, everything is written to a single `index.mdoc`:

```
blog/what-a-day/
└── index.mdoc
```

The `index.mdoc` file:

```markdown
---
title: What a day
publishedDate: 2023-07-27
---

What a **beautiful** day!

## Let's go to the beach

I say we pack our swimmers and towels and head to the beach.

Who's with me?
```

<Tip>
  Single-file output is the most common format for blog posts and documentation pages, as it mirrors the `.md`/`.mdx` conventions used by most static site generators.
</Tip>

## Options reference

<ParamField path="format.data" type="'yaml' | 'json'" default="'yaml'">
  The format used for the data/frontmatter portion of each entry. Defaults to `'yaml'`.
</ParamField>

<ParamField path="format.contentField" type="string">
  The schema key of a `markdoc`, `mdx`, or `document` field to use as the file body. When set, all other fields are written as frontmatter in the same file rather than in a separate data file.
</ParamField>
