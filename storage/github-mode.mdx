---
title: "GitHub mode"
description: "Connect Keystatic to a GitHub repository for branch-based editing, pull request workflows, and team collaboration."
---

GitHub mode connects Keystatic to a GitHub repository. Editors log in with GitHub, make changes on branches, and merge via pull requests — the same workflow developers use for code.

Your project must already be hosted on GitHub, and collaborators need `write` access to the repository.

<Note>
  If you'd prefer not to manage a GitHub OAuth app yourself, [Keystatic Cloud](/cloud) provides managed authentication with no manual setup.
</Note>

## Configuration

Set `storage.kind` to `'github'` and specify the repository in your `keystatic.config.ts`:

```typescript keystatic.config.ts
import { config } from '@keystatic/core';

export default config({
  storage: {
    kind: 'github',
    repo: {
      owner: 'your-org',
      name: 'your-repo',
    },
  },
  // ...collections and singletons
});
```

You can also pass the repository as a single `'owner/name'` string:

```typescript keystatic.config.ts
export default config({
  storage: {
    kind: 'github',
    repo: 'your-org/your-repo',
  },
  // ...
});
```

## Setting up GitHub mode

<Steps>
  <Step title="Update your Keystatic config">
    Change `storage.kind` from `'local'` to `'github'` and add your repository details, as shown above.
  </Step>

  <Step title="Create a GitHub App">
    Visit `/keystatic` in your browser. You will be prompted to log in with GitHub.

    The first time you click **Login with GitHub**, Keystatic walks you through creating a GitHub App for your project. Choose a name for the app and click **Create GitHub App**.

    If you know your deployed project URL or your repository is owned by a GitHub organization, you can fill those fields in before proceeding — otherwise leave them blank.
  </Step>

  <Step title="Grant repository access">
    After creating the app, GitHub will prompt you to install it and grant it access to your repository. Select the repository and confirm the permissions.

    Once complete, you'll be redirected back to the Keystatic Admin UI — now running in GitHub mode. You can tell by the branch dropdown that appears in the top bar.
  </Step>

  <Step title="Check the generated environment variables">
    Keystatic automatically writes the required credentials to a `.env` file in your project root:

    ```bash .env
    KEYSTATIC_GITHUB_CLIENT_ID=...
    KEYSTATIC_GITHUB_CLIENT_SECRET=...
    KEYSTATIC_SECRET=...
    # Next.js:
    NEXT_PUBLIC_KEYSTATIC_GITHUB_APP_SLUG=...
    # Astro:
    # PUBLIC_KEYSTATIC_GITHUB_APP_SLUG=...
    ```

    These variables authenticate editors based on their GitHub repository access. Any user with `write` access to the repository can log in and use the Admin UI.
  </Step>

  <Step title="Copy variables to your deployed environment">
    When deploying, copy the same environment variables to your hosting provider. Keystatic's API routes must run on a Node.js runtime — Vercel and similar platforms work out of the box.

    <Warning>
      Never commit your `.env` file to the repository. Add it to `.gitignore` if it isn't there already.
    </Warning>
  </Step>
</Steps>

## Branch prefix

The `branchPrefix` option limits which branches Keystatic shows and creates:

```typescript keystatic.config.ts
export default config({
  storage: {
    kind: 'github',
    repo: 'your-org/your-repo',
    branchPrefix: 'content/',
  },
  // ...
});
```

With this config, Keystatic only lists branches that start with `content/` and only allows creating new branches with that prefix. This is useful for keeping content branches separate from feature or release branches.

## Adding a redirect URI

If GitHub returns an error like `The redirect_uri is not associated with this application` when you try to authenticate on a deployed environment, you need to add the callback URL to your GitHub App's settings.

<Steps>
  <Step title="Open your GitHub App settings">
    - **Personal account**: go to `https://github.com/settings/apps/<app-slug>`
    - **Organization**: go to `https://github.com/organizations/<org-name>/settings/apps/<app-slug>`
  </Step>

  <Step title="Add the callback URL">
    Under **Callback URL**, click **Add Callback URL** and enter:

    ```
    https://your-domain.com/api/keystatic/github/oauth/callback
    ```

    Click **Save changes**.
  </Step>

  <Step title="Retry authentication">
    Reload the Keystatic login page. The error should no longer appear.
  </Step>
</Steps>

## Environment variables reference

```bash .env.example
# GitHub App credentials (generated by Keystatic setup)
KEYSTATIC_GITHUB_CLIENT_ID=
KEYSTATIC_GITHUB_CLIENT_SECRET=

# Random secret used to sign session tokens — generate with: openssl rand -hex 32
KEYSTATIC_SECRET=

# GitHub App slug (choose the variable name for your framework)
NEXT_PUBLIC_KEYSTATIC_GITHUB_APP_SLUG=   # Next.js
PUBLIC_KEYSTATIC_GITHUB_APP_SLUG=        # Astro
```
